# JavaScript Basics – The Language of the Web

## Introduction

JavaScript (JS) is the programming language of the web. While HTML provides the **structure** of a webpage and CSS provides the **appearance**, JavaScript adds **behavior and interactivity**.

Think of a modern website like a human body:

| Human Body | Web Development |
|------------|-----------------|
| Skeleton | HTML |
| Skin & Clothes | CSS |
| Brain & Muscles | JavaScript |

Without JavaScript, websites would only display static content. With JavaScript, websites can respond to user actions, update content dynamically, communicate with servers, and build complete web applications.

- Frontend Development
- Backend Development (Node.js)
- Mobile App Development (React Native)
- Desktop Applications (Electron)
- Game Development
- Artificial Intelligence Interfaces
- Browser Extensions

Today, JavaScript is used almost everywhere.

# Variables

## What is a Variable?

A variable is a container that stores information.

Think of variables as labeled storage boxes.

```text
+-------------+
| Name        |
|-------------|
| Tauseef     |
+-------------+

+-------------+
| Age         |
|-------------|
| 39          |
+-------------+

+-------------+
| City        |
|-------------|
| Bangalore   |
+-------------+
```

Instead of remembering values directly, we give them names.

---

# Declaring Variables

JavaScript provides three ways to declare variables.

## 1. var

```javascript
var name = "Tauseef";
```

Characteristics:

- Function scoped
- Can be redeclared
- Can be reassigned
- Hoisted

Example:

```javascript
var city = "Bangalore";

city = "Mumbai";

console.log(city);
```

Output:

```text
Mumbai
```

---

## 2. let

Modern JavaScript recommends using `let` for values that may change.

```javascript
let age = 39;

age = 40;
```

Characteristics:

- Block scoped
- Cannot be redeclared
- Can be reassigned

---

## 3. const

Used for values that should never change.

```javascript
const country = "India";
```

Attempting to change it:

```javascript
country = "USA";
```

Output:

```text
TypeError
```

---

# var vs let vs const

| Feature | var | let | const |
|----------|-----|-----|-------|
| Scope | Function | Block | Block |
| Redeclare | ✅ | ❌ | ❌ |
| Reassign | ✅ | ✅ | ❌ |
| Hoisted | ✅ | ✅ (Temporal Dead Zone) | ✅ (Temporal Dead Zone) |

# Scope

Scope determines where a variable is accessible.

---

## Global Scope

Accessible everywhere.

```javascript
let company = "Accenture";

function display(){

    console.log(company);

}
```

---

## Function (Local) Scope

Variable exists only inside the function.

```javascript
function display(){

    let city = "Bangalore";

    console.log(city);

}
```

Outside:

```javascript
console.log(city);
```

Output:

```text
ReferenceError
```

---

## Block Scope

Variables declared using `let` and `const` exist only inside `{}`.

```javascript
if(true){

    let score = 90;

}
```

Outside:

```javascript
console.log(score);
```

Output:

```text
ReferenceError
```

---

# Hoisting

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution.

---

## var Hoisting

```javascript
console.log(name);

var name = "Tauseef";
```

Internally becomes:

```javascript
var name;

console.log(name);

name = "Tauseef";
```

Output:

```text
undefined
```

---

## let and const Hoisting

They are hoisted but cannot be accessed before declaration because of the **Temporal Dead Zone (TDZ)**.

```javascript
console.log(age);

let age = 39;
```

Output:

```text
ReferenceError
```

# Control Flow Statements

Control flow determines which code executes based on conditions.

---

## if Statement

```javascript
let age = 20;

if(age >= 18){

    console.log("Adult");

}
```

---

## if...else

```javascript
let age = 15;

if(age >= 18){

    console.log("Adult");

}
else{

    console.log("Minor");

}
```

Output:

```text
Minor
```

---

## else if

Useful when multiple conditions exist.

```javascript
let marks = 82;

if(marks >= 90){

    console.log("Grade A");

}
else if(marks >= 75){

    console.log("Grade B");

}
else if(marks >= 50){

    console.log("Grade C");

}
else{

    console.log("Fail");

}
```

---

## switch Statement

Useful for fixed values.

```javascript
let role = "admin";

switch(role){

    case "admin":

        console.log("Administrator");

        break;

    case "student":

        console.log("Student");

        break;

    default:

        console.log("Guest");

}
```

---

## Ternary Operator

Short form of `if...else`.

Syntax:

```javascript
condition
    ? valueIfTrue
    : valueIfFalse;
```

Example:

```javascript
let age = 20;

let status =
    age >= 18
    ? "Adult"
    : "Minor";

console.log(status);
```

Output:

```text
Adult
```

# Functions – Building Blocks of JavaScript

## Introduction

Functions are one of the most fundamental concepts in JavaScript.

A **function** is a reusable block of code that performs a specific task.

Instead of writing the same code repeatedly, you can write it once inside a function and call it whenever needed.

---

# Why Functions are Important

Imagine you're building a **Learning Management System (LMS)**.

Without functions:

```javascript
console.log("Welcome Tauseef");
console.log("Welcome John");
console.log("Welcome Sarah");
console.log("Welcome Ahmed");
```

As your application grows, this becomes repetitive.

With functions:

```javascript
function welcomeStudent(name) {
    console.log(`Welcome ${name}`);
}

welcomeStudent("Tauseef");
welcomeStudent("John");
welcomeStudent("Sarah");
welcomeStudent("Ahmed");
```

Advantages:

- Reusable
- Easy to maintain
- Less code duplication
- Better organization
- Easier debugging

---

# Real-World Analogy

Think of a **coffee vending machine**.

```text
Press Button
      ↓
Machine Executes Steps
      ↓
Coffee is Ready
```

Similarly:

```text
Call Function
      ↓
JavaScript Executes Code
      ↓
Returns Result
```

A function is like a machine that performs a task whenever you ask it to.

# Closures

Closures are one of the most powerful JavaScript concepts.

A closure allows an inner function to remember variables from its outer function even after the outer function has finished executing.

---

## Example

```javascript
function createCounter() {

    let count = 0;

    return function() {

        count++;

        return count;

    };

}
```

Calling:

```javascript
const counter = createCounter();

console.log(counter());
console.log(counter());
console.log(counter());
```

Output:

```text
1
2
3
```

---

# Why Does This Work?

Execution:

```text
createCounter()

↓

count = 0

↓

Returns Inner Function

↓

Outer Function Ends

↓

Inner Function Still Remembers count
```

This memory is called a **closure**.

---

# Real-World Analogy

Imagine a locker.

```text
Outer Function

↓

Places Data Inside Locker

↓

Leaves

↓

Inner Function Still Has Locker Key
```

Even though the outer function has finished, the inner function still has access to the stored data.

---

# Closures in Real Applications

Closures are commonly used in:

- Private variables
- Event handlers
- Callbacks
- Timers
- Module pattern
- React Hooks

# Arrays and Objects – Working with Data

Arrays and Objects are two of the most important data structures in JavaScript.

# Asynchronous JavaScript – Handling Time-sensitive Code

## Introduction

JavaScript is **single-threaded**, meaning it executes **one task at a time** using a single call stack.

But modern applications perform many operations that take time, such as:

- Fetching data from an API
- Reading files
- Uploading images
- Accessing a database
- Waiting for user input
- Waiting for timers

If JavaScript waited for each task to complete before moving to the next one, web applications would become slow and unresponsive.

To solve this, JavaScript uses **Asynchronous Programming**.

---

# What is Asynchronous Programming?

Asynchronous programming allows JavaScript to start a long-running task and continue executing other code without waiting for the task to finish.

---

## Real-World Analogy

Imagine you're at a restaurant.

### Synchronous

```text
Order Food
      ↓
Wait 20 Minutes
      ↓
Eat
      ↓
Pay Bill
```

You cannot do anything while waiting.

---

### Asynchronous

```text
Order Food
      ↓
Talk with Friends
      ↓
Check Mobile
      ↓
Food Arrives
      ↓
Eat
```

You continue doing other work while waiting.

JavaScript behaves similarly.

---

# Why Asynchronous Programming is Important

Without asynchronous programming:

```text
Request Database
      ↓
Wait
      ↓
Wait
      ↓
Wait
      ↓
Continue
```

Application freezes.

---

With asynchronous programming:

```text
Request Database
      ↓
Continue Other Tasks
      ↓
Database Responds
      ↓
Process Result
```

Application remains responsive.

---

Nested callbacks become difficult to read.

```text

            ↓

getAssignments()

                ↓

```

This pyramid shape is called **Callback Hell**.

Problems:

- Hard to read
- Difficult to debug
- Difficult to maintain

# JavaScript and the DOM – Interacting with the Browser

## Introduction

Until now, we've learned how JavaScript works with:

- Variables
- Functions
- Arrays
- Objects
- Asynchronous Programming

But none of these topics actually change what users see on a webpage.

To make a webpage interactive, JavaScript needs a way to communicate with HTML.

That's where the **DOM (Document Object Model)** comes in.

---

# What is the DOM?

DOM stands for:

```text
Document Object Model
```

The DOM is a programming interface that represents an HTML page as a tree of JavaScript objects.

It allows JavaScript to:

- Read HTML
- Modify HTML
- Change CSS
- Respond to user actions
- Add new elements
- Remove existing elements

Without the DOM, JavaScript cannot interact with a webpage.

```text
      ├── header
      ├── section
      ├── article
      └── footer
```

JavaScript can walk through this tree and modify any node.

# Common Beginner Mistakes

## Using `innerHTML` for User Input

Avoid:

```javascript
element.innerHTML =
userInput;
```

Can introduce security risks such as Cross-Site Scripting (XSS).

Prefer:

```javascript
element.textContent =
userInput;
```

when displaying plain text.

---

## Forgetting `querySelectorAll()` Returns Multiple Elements

Incorrect:

```javascript
const items =
document.querySelectorAll("li");

items.style.color =
"red";
```

Correct:

```javascript
items.forEach(item => {

    item.style.color =
    "red";

});
```

---

## Accessing Elements Before They Exist

Incorrect:

```javascript
const heading =
document.getElementById("title");
```

before HTML loads.

Solution:

```javascript
<script src="script.js" defer></script>
```

or place the `<script>` tag just before the closing `</body>` tag.

---

# Best Practices

- Prefer `querySelector()` for flexible selection.
- Use `textContent` when displaying plain text.
- Use `classList` instead of directly changing many inline styles.
- Avoid excessive use of `innerHTML`.
- Keep DOM updates minimal for better performance.

# Event Handling – Making Web Pages Interactive

## Introduction

Until now, we've learned how to:

- Create HTML pages
- Style them with CSS
- Access and modify them using the DOM

But web pages are still **static** until users interact with them.

Questions like:

- What happens when a button is clicked?
- How does a login form submit?
- How does a menu open?
- How does a search box respond while typing?

The answer is **Events**.

Events allow JavaScript to respond to user actions and make webpages interactive.

---

# What is an Event?

An **event** is an action or occurrence that happens in the browser.

It can be triggered by:

- User actions
- Browser actions
- System events

# What is Object-Oriented Programming (OOP)?

Object-Oriented Programming is a programming paradigm that organizes code around **objects** rather than individual functions.

Objects contain:

- Data (Properties)
- Behavior (Methods)

Instead of thinking:

We think:


# Inheritance

Inheritance allows one class to inherit properties and methods from another.

Example:

```text
Person

↓

Student

↓

Instructor
```

---

# Parent Class

```javascript
class Person {

    constructor(name){

        this.name = name;

    }

}
```

---

# Child Class

```javascript
class Student

extends Person {

}
```

Now Student inherits everything from Person.

---

# Using `super()`

The parent constructor is called using:

```javascript
super()
```

Example:

```javascript
class Student

extends Person {

    constructor(

        name,

        course

    ){

        super(name);

        this.course = course;

    }

}
```

---

Creating:

```javascript
const student =
new Student(

    "Tauseef",

    "JavaScript"

);
```

Output:

```text
Name

↓

Tauseef

Course

↓

JavaScript
```

---

# Method Inheritance

Parent:

```javascript
class Person {

    greet(){

        console.log(

            "Hello"

        );

    }

}
```

Child:

```javascript
class Student

extends Person {

}
```

Usage:

```javascript
const student =
new Student();

student.greet();
```

Output:

```text
Hello
```

Student automatically inherits `greet()`.

# Method Overriding

Child classes can replace inherited methods.

```javascript
class Student

extends Person {

    greet(){

        console.log(

            "Welcome Student"

        );

    }

}
```

Now:

```javascript
student.greet();
```

Output:

```text
Welcome Student
```

# Prototype Chain

Every JavaScript object has a prototype.

When JavaScript cannot find a property, it searches upward.

Example:

```text
student

↓

Student Prototype

↓

Person Prototype

↓

Object Prototype

↓

null
```

This is called the **Prototype Chain**.

---

# Why Prototypes?

Without prototypes:

```text
Every Object

↓

Own Copy of Methods
```

Huge memory usage.

---

With prototypes:

```text
All Objects

↓

Share Same Methods
```

Much more efficient.

---

# Visual Representation

```text

↓

Student Prototype

↓

Person Prototype

↓

Object Prototype

↓

null
```

Polymorphism means:

One interface.

Many implementations.

Example:

Student


Instructor


# Encapsulation

Encapsulation means keeping related data and methods together while hiding internal implementation details.

Example:

```javascript
class Student {

    constructor(name){

        this.name = name;

    }

    login(){

        console.log(

            `${this.name} Logged In`

        );

    }

}
```

The object's data and behavior are bundled together.

---

# Private Fields (Modern JavaScript)

```javascript
class Student {

        password;

    }

}
```

Outside the class:

```javascript
student.#password
```

Output:

```text
SyntaxError
```

The field is private and cannot be accessed directly.

# Part 1 – Closures

---

# What is a Closure?

A **closure** is created when an inner function remembers variables from its outer function even after the outer function has finished executing.

This is one of the most powerful features of JavaScript.

---

## Real-World Analogy

Imagine a locker.

```text
Teacher

↓

Puts Notes Inside Locker

↓

Leaves School

↓

Student Still Has Locker Key

↓

Can Read Notes Anytime
```

The locker is like a closure.

The outer function is gone, but the inner function still remembers the data.

---

# Example

```javascript
function createCounter() {

    let count = 0;

    return function () {

        count++;

        return count;

    };

}

const counter = createCounter();

console.log(counter());

console.log(counter());

console.log(counter());
```

Output

```text
1

2

3
```

---

# Why Does It Work?

Execution

```text
createCounter()

↓

count = 0

↓

Returns Inner Function

↓

Outer Function Ends

↓

Inner Function Still Remembers count
```

This memory is called a **Closure**.

---

# Multiple Closures

```javascript
const counter1 =
createCounter();

const counter2 =
createCounter();
```

Each counter has its own private memory.

Output

```text
counter1()

1

counter1()

2

counter2()

1
```

---

# Practical Uses

Closures are commonly used for:

- Private variables
- React Hooks
- Event handlers
- Timers
- Memoization
- Factory functions

---

# Example – Private Variable

```javascript
function createStudent(name) {

    let marks = 90;

    return {

        getMarks() {

            return marks;

        }

    };

}

const student =
createStudent("Tauseef");

console.log(
student.getMarks()
);
```

Output

```text
90
```

Notice that:

```javascript

```

cannot be accessed directly.

# Part 2 – Lexical Scoping

---

# What is Lexical Scope?

Lexical means:

```text
Where the function was written
```

Not:

```text
Where the function is called
```

JavaScript determines scope **during code creation**, not during execution.

---

# Example

```javascript
let language =
"JavaScript";

function outer() {

    function inner() {

        console.log(language);

    }

    inner();

}

outer();
```

Output

```text
JavaScript
```

---

# Scope Chain

```text
Inner Function

↓

Outer Function

↓

Global Scope

↓

null
```

JavaScript searches upward until it finds the variable.

---

# Example

```javascript
let company =
"OpenAI";

function one() {

    let city =
    "Bangalore";

    function two() {

        console.log(company);

        console.log(city);

    }

    two();

}

one();
```

Output

```text
OpenAI

Bangalore
```

---

# Lexical Scope + Closure

Closures work because JavaScript remembers the lexical environment.

```text
Outer Scope

↓

Inner Function

↓

Closure Created
```

# Part 3 – Understanding `this`

---

# What is `this`?

`this` refers to **the object that is executing the current function**.

The value of `this` depends on **how a function is called**, not where it is defined.

This is one of the most confusing concepts in JavaScript.

---

# Global Context

Browser

```javascript
console.log(this);
```

Output

```text
window
```

---

Node.js

```javascript
console.log(this);
```

Output

```text
{}
```

---

# Inside an Object

```javascript
const student = {

    name: "Tauseef",

    greet() {

        console.log(this.name);

    }

};

student.greet();
```

Output

```text
Tauseef
```

Here:

```javascript
this
```

refers to:

```javascript
student
```

---

# Inside a Regular Function

```javascript
function greet() {

    console.log(this);

}

greet();
```

In strict mode:

```text
undefined
```

Without strict mode (browser):

```text
window
```

---

# Inside an Arrow Function

Arrow functions do **not** create their own `this`.

They inherit `this` from the surrounding scope.

Example

```javascript
const student = {

    name: "Tauseef",

    greet() {

        const show = () => {

            console.log(this.name);

        };

        show();

    }

};

student.greet();
```

Output

```text
Tauseef
```

---

# Regular Function vs Arrow Function

| Regular Function | Arrow Function |
|------------------|----------------|
| Own `this` | Uses parent's `this` |
| Dynamic | Lexical |
| Used for object methods | Great for callbacks |

# Part 4 – call(), apply(), bind()

---

Sometimes we want to manually decide what `this` should be.

JavaScript provides:

```text
call()

apply()

bind()
```

---

# call()

Immediately invokes a function.

```javascript
function greet() {

    console.log(this.name);

}

const student = {

    name: "Tauseef"

};

greet.call(student);
```

Output

```text
Tauseef
```

---

# Passing Parameters

```javascript
function greet(course) {

    console.log(

        this.name,

        course

    );

}

greet.call(

    student,

    "React"

);
```

Output

```text
Tauseef React
```

---

# apply()

Works like `call()`.

Difference:

Arguments are passed as an array.

```javascript
greet.apply(

    student,

    ["React"]

);
```

---

# call() vs apply()

| call() | apply() |
|----------|----------|
| Separate arguments | Array of arguments |

---

# bind()

Unlike `call()` and `apply()`, `bind()` does **not** execute immediately.

It returns a new function.

```javascript
const greetStudent =
greet.bind(student);

greetStudent();
```

Output

```text
Tauseef
```

---

# Why bind()?

Very common in:

- React
- Event handlers
- Timers
- Callbacks

---

# Summary

```text
call()

↓

Runs Immediately

apply()

↓

Runs Immediately

bind()

↓

Returns New Function
```

# What is Destructuring?

Destructuring allows us to extract values from arrays and objects into variables.

Instead of:

```javascript
const student = {

    name: "Tauseef",

    course: "JavaScript"

};

const name =
student.name;

const course =
student.course;
```

We can write:

```javascript
const {

    name,

    course

} = student;
```

---

# Real-World Analogy

Imagine unpacking a suitcase.

Without destructuring:

```text
Open Suitcase

↓

Take Shirt

↓

Take Shoes

↓

Take Laptop
```

With destructuring:

```text
Unpack Everything At Once
```

---

# Array Destructuring

```javascript
const courses = [

    "HTML",

    "CSS",

    "JavaScript"

];

    first,

    second,

    third
```

```text
first

↓

HTML

second

↓

CSS

third

↓

JavaScript
```

# Skipping Values

```javascript

    html,

    ,
```

```text

↓

```

# Part 8 – WeakMap

---

# What is WeakMap?

A WeakMap is similar to Map but:

- Keys must be objects.
- Keys are weakly referenced.
- Automatically cleaned by the Garbage Collector.

Example

```javascript
const users =
new WeakMap();

const student = {};

users.set(

    student,

    "Tauseef"

);
```

---

# When to Use

Useful for:

- Private object data
- Caching
- Internal framework implementations

---

# Part 9 – WeakSet

---

# What is WeakSet?

Similar to Set.

Differences:

- Stores only objects.
- Objects can be garbage collected.
- Cannot iterate.

Example

```javascript
const activeUsers =
new WeakSet();

const student = {};

activeUsers.add(

    student

);
```

# Set vs Array

| Array | Set |
|--------|-----|
| Allows Duplicates | Unique Values Only |
| Indexed | Not Indexed |
| Many Methods | Optimized for Uniqueness |

# Best Practices

- Use destructuring to simplify code.
- Prefer template literals over string concatenation.
- Use default parameters to avoid `undefined`.
- Use rest parameters for flexible functions.
- Use spread for copying and merging arrays or objects.
- Use `Map` when keys are not simple strings.
- Use `Set` to remove duplicate values.
- Use `WeakMap` and `WeakSet` for object-specific metadata and memory-efficient storage.
